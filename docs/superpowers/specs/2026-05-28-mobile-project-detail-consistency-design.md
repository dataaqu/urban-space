# მობილურის პროექტის შიდა გვერდის თანმიმდევრულობა

თარიღი: 2026-05-28
სტატუსი: დამტკიცებული დიზაინი (implementation plan მოსალოდნელია)

## პრობლემა

პროექტის შიდა გვერდი (`/[locale]/projects/[id]`) **მობილურზე სხვადასხვა მოწყობილობაზე
სხვადასხვაგვარად ჩანს** — მაგ. iPhone 14-ზე ერთნაირად, iPhone 17-ზე სხვანაირად.
სასურველია **ერთი, თანმიმდევრული განლაგება ყველა ტელეფონზე**.

გვერდს აქვს ორი ვიზუალური სტილი:
- **1-ფოტოიანი** — ერთი დიდი ფოტო.
- **2-ფოტოიანი** — ორი ფოტო, მობილურზე ვერტიკალურად დაწყობილი.

სტრუქტურა (ზემოდან ქვემოთ): Close ღილაკი → ფოტო(ები) → (მხოლოდ პირველ slide-ზე)
სათაური + აღწერა → `Project Information` ღილაკი. მარჯვენა კიდეზე pagination dots.

## Root cause

`src/components/projects/ProjectDetailClient.tsx`-ში მობილური (base) განლაგება იყენებს:

1. **`justify-center`** — კონტენტი ვერტიკალურად ცენტრშია, ხოლო Close ღილაკი
   ზემოდან **ფიქსირებულ** წერტილზეა (`fixed top-[72px]`). სხვადასხვა სიმაღლის
   ეკრანზე ცენტრი სხვა ადგილას ჩამოდის → Close↔ფოტოს მანძილი იცვლება.
2. **ფიქსირებული `vh` სიმაღლეები** — `h-[64vh]` (2 ფოტო) / `h-[46vh]` (1 ფოტო),
   რომლებიც px გადატანებთან (`mt-[60px]`, `pt-[52px]`) ერთად თითო ეკრანის
   პროპორციაზე სხვაგვარად ჯდება.
3. iOS Safari-ს `dvh`/toolbar-ის ქცევა iOS ვერსიებს შორის განსხვავდება, რაც
   ფიქსირებულ-`vh` განლაგებას კიდევ უფრო არასტაბილურს ხდის.

**მნიშვნელოვანი დაკვირვება:** ტაბლეტი (`md:`) და desktop (`lg:`) **უკვე** იყენებს
სწორ, ზემოდან-მიმაგრებულ `flex-1` განლაგებას (`md:justify-start md:flex-1
md:min-h-0`). ცურვა **მხოლოდ მობილურ (base) კლასებშია**.

## გადაწყვეტა (Approach A)

მობილურმა (base) გადაიღოს **ზუსტად იგივე ზემოდან-მიმაგრებული `flex-1` სტრუქტურა,
რომელსაც ტაბლეტი უკვე იყენებს**. ვიზუალური სტილი (ორივე: 1- და 2-ფოტოიანი) უცვლელი
რჩება; იცვლება მხოლოდ ვერტიკალური განლაგების მექანიკა, რათა ფოტოს არემ თავად
ალაგოს მოწყობილობებს შორის სიმაღლის სხვაობა.

### ზუსტი ცვლილებები (მხოლოდ base კლასები)

ფაილი: `src/components/projects/ProjectDetailClient.tsx`

1. **ცენტრალური სვეტის container** (ამჟამად `... justify-center ... md:justify-start ...`):
   base `justify-center` → `justify-start`. `md:`/`lg:` და `short-landscape:`
   კლასები უცვლელი.

2. **ფოტოს არე (image stage):** ამჟამინდელი base პირობითი სიმაღლე
   `hasTwoImages ? 'h-[64vh]' : 'h-[46vh]'` → `flex-1 min-h-0` (იგივე რასაც
   `md:h-auto md:flex-1 md:min-h-0` უკვე აკეთებს ტაბლეტზე). პირობითი `vh` ლოგიკა
   იხსნება.

3. **2-ფოტოიანი მობილური ბლოკი** (ამჟამად `mt-[60px] flex h-full ... flex-col
   ... gap-3 md:hidden`): იხსნება `mt-[60px]`. ბლოკი რჩება ვერტიკალურად
   დაწყობილი (`flex-col`, `gap-3`, `justify-center`), `h-full`-ით ავსებს `flex-1`
   არეს; თითო ფოტო `object-contain`, `max-h-[48%]` (ამჟამინდელი `max-h-[47%]`
   შეიძლება დარჩეს — საბოლოო რიცხვი implementation-ში დაზუსტდება ისე, რომ ორი
   ფოტო არასდროს გადაიჭიმოს არედან).

4. **სათაური/აღწერის არე და INFO ღილაკი:** **არ იცვლება.** უკვე ფიქსირებული
   სიმაღლეებია (`min-h-[80px]`, ღილაკი `h-[60px]`) → ღილაკის პოზიცია ისედაც
   სტაბილურია.

### საბოლოო მობილური სტრუქტურა (flex column, ზემოდან ქვემოთ)

```
[ pt (ფიქსირებული, Close ღილაკის ქვემოთ) ]
[ ფოტო(ები)  — flex-1, object-contain, ივსება ]
[ სათაური + აღწერა — min-h-[80px] (მხოლოდ slide 0-ზე) ]
[ INFO ღილაკი — h-[60px] ]
[ pb (ფიქსირებული) ]
```

ფოტოს `flex-1` არე შთანთქავს ეკრანის სიმაღლის სხვაობას → Close, სათაური და ღილაკი
ერთსა და იმავე ლოგიკით ეწყობა ყველა ტელეფონზე, ფოტო კი ცოცხლად ერგება ზომას.

## მკაცრი ფარგლები (Scope)

**იცვლება მხოლოდ:**
- ფაილი: `src/components/projects/ProjectDetailClient.tsx`
- კლასები: **მხოლოდ მობილური (base, პრეფიქსის გარეშე)**.

**არ იცვლება (regression აკრძალულია):**
- ტაბლეტი (`md:`, 768–1023px) — უკვე სწორია.
- desktop (`lg:`, ≥1024px) — drawer მარცხნიდან, განლაგება, ყველაფერი.
- `short-landscape:` ვარიანტი.
- pagination dots (`top-[42%]` — ისედაც პროპორციულია).
- slide navigation (wheel/touch/keyboard), info drawer-ის ლოგიკა, ESC/scroll-lock.
- ნებისმიერი სხვა გვერდი ან კომპონენტი.

## მისაღები შედეგის კრიტერიუმი

- iPhone 14, iPhone 15/16/17 (და ზოგადად 360–430px სიგანის ტელეფონები)
  პროექტის შიდა გვერდზე **იდენტურ განლაგებას** აჩვენებენ: Close → ფოტო → (slide 0)
  სათაური → INFO ღილაკი, ერთი და იმავე ლოგიკით.
- ორივე სტილი (1- და 2-ფოტოიანი) ვიზუალურად ისეთივეა, როგორიც ახლა.
- ტაბლეტი და desktop **ბაიტ-ბაიტ უცვლელია**.
